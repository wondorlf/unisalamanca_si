MÓDULO 1: SISTEMAS

Concepto de Sistema

"Un sistema es un conjunto de elementos interrelacionados para lograr un mismo objetivo."— Ludwig von Bertalanffy (1945)

Análisis y Diseño

📊 Análisis de Sistemas

Estudiar procedimientos para identificar QUÉ debe hacer el sistema.

  • Identificar problemas existentes
  • Recopilar datos y requisitos
  • Evaluar soluciones actuales
  • Documentar el sistema actual

Análisis: ¿QUÉ hacer?  →  Diseño: ¿CÓMO hacerlo?

✏️ Diseño de Sistemas

Especificar arquitectura, componentes, módulos, interfaces y datos.

  • Definir arquitectura del sistema
  • Diseñar módulos y componentes
  • Crear prototipos
  • Planificar la implementación

🔧 Características de los Sistemas

  • Holismo: El todo es mayor que la suma de partes
  • Equifinalidad: Diversos caminos para un mismo fin
  • Homeostasis: Capacidad de autorregulación
  • Adaptación: Se ajustan al entorno
  • Entropía: Tendencia al desorden

📦 Tipos de Sistemas

  • Naturales: Sistema solar, ecosistema
  • Artificiales: Sistema computarizado
  • Abiertos: Empresa, organismo vivo
  • Cerrados: Reloj mecánico
  • Conceptuales: Teoría matemática

MÓDULO 2: INFORMACIÓN

Definición de Información

📖 Teoría de la Información (Shannon, 1948)

"La información es la medida de la reducción de la incertidumbre que proporciona un mensaje."

🏢 En Ingeniería de Sistemas

"La información son datos procesados y organizados que tienen significado y utilidad para la toma de decisiones."

Características de la Información

⏱️ Oportunidad

Disponible cuando se necesita. Información tardía puede ser inútil.

🎯 Precisión

Correcta y confiable. Errores llevan a malas decisiones.

📝 Relevancia

Pertinente al propósito para el que se necesita.

🔍 Comprensibilidad

Clara y fácil de interpretar por el usuario.

📦 Completitud

Incluye todos los datos necesarios para la decisión.

🔄 Accesibilidad

Obtenida fácilmente por quienes la necesitan.

MÓDULO 3: SISTEMAS DE INFORMACIÓN

Definición

"Un sistema de información es el conjunto de agentes, códigos y procesos que interactúan coordinadamente entre sí con un fin común o propósito."
🖥️

Hardware

Equipo físico para procesar y almacenar datos.

💿

Software

Programas y procedimientos de procesamiento.

📊

Datos

Información que representa las actividades.

🌐

Red

Infraestructura de comunicación.

👥

Personas

Desarrollan, mantienen y usan el sistema.

📋

🔄 Operaciones Principales

  • Input: Capturar datos crudos
  • Proceso: Calcular, clasificar, ordenar
  • Almacenamiento: Guardar en BD o archivos
  • Output: Generar reportes, pantallas
  • Feedback: Evaluar y corregir

📋 Ciclo de Vida del SI

  1. Conocer la organización
  2. Identificar problemas y oportunidades
  3. Diagnóstico
  4. Determinar necesidades
  5. Propuesta
  6. Diseño del sistema
  7. Codificación
  8. Implementación
  9. Mantenimiento

Tipos de Sistemas de Información

Modelo de la Pirámide Organizacional

🏢 EIS - Información Ejecutiva
📊 DSS - Soporte a Decisiones
📈 MIS - Información Gerencial
📋 TPS - Procesamiento de Transacciones
⚙️ OAS - Automatización de Oficinas

📋 TPS

Registra operaciones diarias: ventas, inventario, nómina. Operativo

📈 MIS

Genera reportes periódicos para gerentes. Táctico

📊 DSS

Análisis de escenarios y simulaciones. Estratégico

🏢 EIS

Información de alto nivel para C-suite. Estratégico

Otros Tipos de Sistemas de Información

🤖 Sistema Experto (SE)

Emula el comportamiento de un experto humano con inteligencia artificial y reglas de inferencia.

📦 ERP

Enterprise Resource Planning. Integra toda la información y procesos de una organización.

🌐 SIG

Sistema de Información Geográfica. Análisis de información georreferenciada.

📝 OAS

Sistemas de Automatización de Oficinas. Optimizan el trabajo administrativo.

🏢 SI Estratégico

Uso de TI para respaldar la estrategia competitiva de la organización.

☁️ SI en la Nube

Sistemas accesibles vía internet. Permiten trabajo remoto y escalabilidad.

Información y Toma de Decisiones

📊 Niveles de Decisión

🏢 Estratégica (Alta Gerencia)▼

Largo plazo: expansión, fusiones, nuevos mercados. Requiere información externa y estratégica.

📋 Táctica (Gerencia Media)▼

Mediano plazo: asignación de recursos, presupuestos departamentales.

⚙️ Operativa (Supervisores)▼

Corto plazo: horarios, inventario, programación diaria.

💡 Calidad de la Información

  • Relevante: Perteneciente al problema
  • Oportuna: Disponible a tiempo
  • Exacta: Sin errores
  • Completa: Sin omisiones
  • Económica: Costo-beneficio favorable
"Demasiada información puede ser tan dañina como muy poca."

Conclusiones

Conjuntos de elementos interrelacionados para lograr objetivos. Se analizan (QUÉ) y diseñan (CÓMO).

Datos procesados con significado. Su calidad determina la efectividad de las decisiones.
